Summary of HTML5 For Designers by Jeremy Keith

Chapter 1: A brief History of Markup

HTML5 was formed by a bunch of rebels from the W3C who formed Hypertext Application Technology Working Group,(WHATWG).HTML5 replaced XHTML which is now almost dead.

The expected timeline for HTML5 to become a proposed recommendation is said to be 2022. This of course raised outrage from web designers.However the date is actually not important as we can start using parts of the specification as web browsers support those features.

Chapter 2: The Design of HTML5

The design principles

These are key principles that guide the development of the HTML5 development

  1. Support existing content i.e no Year Zero for HTML5.
  2. Backward Compatibility
  3. Keep it simple.
  4. Don't reinvent the wheel.
  5. Pave the cowpaths i.e If it aint broke don't fix it.
  6. Priority of constituencies i.e In case of conflict, consider users over authors over implementers over specifiers over theoretical purity

Chapter 3: Rich Media

HTML5 fills in the holes when in comes to media since it does not require plugins as flash instead the rich media elements in HTML5 are native to the browser. We look at some of the rich media in HTML5

Canvas

The canvas element is an environment for creating dynamic images.The real power of canvas is that its contents can be updated at any moment, drawing new content based on the actions of the user. This ability to respond to user-triggered events makes it possible to create tools and games that would have previously required a plug-in technology such as Flash.

Audio

With HTML5 one can be able to embed an audio. It is very simple to do that. You can also be able to have autoplay, buffering which can all be turned on or off. How cool!

Chapter 5: Semantics

Extensibiliity

Extensibility is about adding more semantic value to elements using the class attribute. This class attribut can be used by browers if they follow the agreed conventions.

Microformats

Microformats are a set of conventions which are agreed upon by a community.They are used with classes to extend the HTML5 attributes. However, they are limited by design. They solve 80% of the use cases with 20% of the effort.

Chapter 6: Using HTML5 Today

  1. Today, not all element in HTML5 are supported by all browsers. However, this does not hinder someone from using them
  2. Browsers don't support default styling for most of the elemnts.
  3. Browsers are also yet to support heading levels of HTML5’s new outline algorithm but are still available for use..
  4. Accessible Rich Internet Applications (ARIA) can be used to support assistive technology.
  5. Validation is a powerful tool but needs to be used wisely.
